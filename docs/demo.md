# DataMind 演示指南

本文档提供一组精心设计的示例问题，用于演示 DataMind 的 Agent 如何**自动调度**不同模块来回答问题。每个问题都标注了预期触发的模块，方便你验证系统是否正常工作。

> 使用方式：启动 Web 界面 (`python server.py`) 或终端 (`python main.py`)，依次输入下面的问题。

---

## 1. RAG 知识库检索 → `knowledge_search`

RAG 模块从 `data/` 目录下的文档中检索语义相关的内容。

| # | 示例问题 | 预期行为 |
|---|---------|---------|
| 1.1 | `DataMind 支持哪些文档格式？` | 检索 sample.txt，返回 PDF/TXT/MD/DOCX 等 |
| 1.2 | `这个项目使用了什么向量数据库？` | 检索到 Chroma 相关描述 |
| 1.3 | `RAG 检索的核心原理是什么？` | 检索到向量化 + 余弦相似度的描述 |
| 1.4 | `LlamaIndex 提供了哪些功能？` | 检索到数据连接器、索引结构、查询接口 |

---

## 2. GraphRAG 图谱检索 → `graph_search`

GraphRAG 模块通过知识图谱的实体和关系进行多跳推理。

| # | 示例问题 | 预期行为 |
|---|---------|---------|
| 2.1 | `DataMind 基于什么框架？` | 图谱推理: DataMind → 基于 → LlamaIndex |
| 2.2 | `DataMind 包含哪些模块？` | 图谱推理: DataMind → 包含模块 → RAG/GraphRAG/NL2SQL |
| 2.3 | `RAG 向量检索使用了什么技术？` | 图谱推理: RAG向量检索 → 使用技术 → Chroma |
| 2.4 | `LlamaIndex 和 Python 是什么关系？` | 图谱推理: LlamaIndex → 是 → Python框架 |

---

## 3. Database 数据库查询 → `database_query`

Database 模块将自然语言转换为 SQL 查询，在 SQLite 中执行。

当前 demo 数据库包含：
- **employees** 表: 8 名员工 (张三、李四、王五、赵六、孙七、周八、吴九、郑十)
- **projects** 表: 4 个项目 (RAG智能助手、数据分析平台、移动端App、品牌推广)

| # | 示例问题 | 预期行为 |
|---|---------|---------|
| 3.1 | `工程部有几个人？` | SQL: SELECT COUNT(*) FROM employees WHERE department='工程部' → 4 |
| 3.2 | `谁的工资最高？` | SQL: SELECT * FROM employees ORDER BY salary DESC LIMIT 1 → 孙七 45000 |
| 3.3 | `北京的员工有哪些？` | SQL: 筛选 city='北京' → 张三、王五、孙七、郑十 |
| 3.4 | `预算超过 20 万的项目有哪些？` | SQL: SELECT * FROM projects WHERE budget > 200000 → RAG智能助手、数据分析平台 |
| 3.5 | `RAG 智能助手项目的负责人是谁？` | SQL: JOIN employees 和 projects → 孙七 |
| 3.6 | `各部门的平均工资是多少？` | SQL: GROUP BY department + AVG(salary) |

---

## 4. Skills 知识型技能 → `skill_search`

Skills 知识模块从 `data/skills/` 下的 Markdown 文件中检索操作流程和最佳实践。

当前 demo 技能文件：
- `数据库运维SOP.md` — 备份、恢复、性能优化、故障排查
- `代码审查指南.md` — 审查流程、审查重点、反馈规范

| # | 示例问题 | 预期行为 |
|---|---------|---------|
| 4.1 | `数据库备份的最佳策略是什么？` | 检索 SOP，返回全量+增量备份策略 |
| 4.2 | `数据库慢查询怎么排查？` | 检索 SOP，返回慢查询分析步骤 |
| 4.3 | `代码审查应该重点关注什么？` | 检索审查指南，返回功能正确性/代码质量/安全性/性能 |
| 4.4 | `Code Review 的反馈应该怎么写？` | 检索审查指南，返回 MUST/SHOULD/NICE 分级 |
| 4.5 | `数据库故障排查的步骤？` | 检索 SOP 故障排查清单 |

---

## 5. Skills 工具型技能 → `calculator` / `get_current_time` / `unit_convert`

工具型技能是 Python 函数，Agent 自动调用获取精确结果。

| # | 示例问题 | 预期行为 |
|---|---------|---------|
| 5.1 | `现在几点了？` | 调用 get_current_time → 返回当前日期时间 |
| 5.2 | `计算 sqrt(144) + 3^4` | 调用 calculator → 12 + 81 = 93 |
| 5.3 | `100 公里等于多少英里？` | 调用 unit_convert → 62.14 mile |
| 5.4 | `25 摄氏度等于多少华氏度？` | 调用 unit_convert → 77°F |

---

## 6. Memory 对话记忆

Memory 模块让 Agent 记住对话上下文。需要连续提问来验证。

| # | 操作 | 预期行为 |
|---|------|---------|
| 6.1 | 先问: `工程部有几个人？` | 正常回答: 4 个 |
| 6.2 | 再问: `他们分别是谁？` | Agent 记得上文在讨论"工程部"，返回张三、李四、孙七、吴九 |
| 6.3 | 再问: `其中谁的工资最高？` | Agent 记得上下文，回答: 孙七 45000 |
| 6.4 | 问: `我刚才问了什么？` | Agent 基于记忆回顾之前的对话内容 |

## 7. 多模块联动 (组合调用)

以下问题可能触发 Agent **同时调用多个工具**，体现智能调度能力。

| # | 示例问题 | 预期触发模块 |
|---|---------|------------|
| 7.1 | `DataMind 用了什么技术栈？各模块分别用了什么？` | RAG + GraphRAG |
| 7.2 | `工资最高的员工负责的是哪个项目？这个项目预算多少，折合多少美元？` | Database + calculator/unit_convert |
| 7.3 | `数据库出了性能问题，应该怎么排查？先帮我看看当前数据库有哪些表` | Skills + Database |
| 7.4 | `今天是几号？帮我算一下如果按全量备份策略，30天前的备份应该从哪天开始保留？` | get_current_time + calculator + Skills |

---

## 8. 普通闲聊 (不触发工具)

以下问题 Agent 会直接用 LLM 回答，不需要调用任何工具。

| # | 示例问题 | 预期行为 |
|---|---------|---------|
| 8.1 | `你好！` | 直接打招呼 |
| 8.2 | `你能做什么？` | 介绍自身能力 |
| 8.3 | `给我讲个笑话` | 直接生成内容 |

## 演示流程建议

如果要做一个完整的演示，推荐按以下顺序：

1. **RAG**: 问 `DataMind 支持哪些文档格式？` → 展示知识库检索
2. **GraphRAG**: 问 `DataMind 包含哪些模块？` → 展示图谱推理
3. **Database**: 问 `谁的工资最高？` → 展示 NL2SQL
4. **Skills**: 问 `数据库备份的最佳策略是什么？` → 展示知识型技能
5. **Tool**: 问 `现在几点了？` → 展示工具调用
6. **Memory**: 问 `工程部有几个人？` 然后问 `他们分别是谁？` → 展示对话记忆
7. **联动**: 问 `工资最高的员工负责哪个项目？预算折合多少美元？` → 展示多模块联动

> 提示：可以打开 Web 界面右侧面板，在提问的同时查看各模块的状态变化。
